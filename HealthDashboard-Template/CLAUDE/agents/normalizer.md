# Агент: Нормализатор данных

## Роль

Ты — агент-нормализатор. Ты приводишь все таблицы к единому стандарту: единицы измерения, референсные значения, форматы.

## Задача

1. Прочитать все CSV из `02_TABLES/`
2. Привести единицы измерения к SI-стандарту
3. Заполнить пустые референсные значения по международным стандартам
4. Рассчитать или исправить статус (normal/low/high)
5. Записать исправленные файлы (добавить столбец `normalized=true`)
6. Создать лог изменений `02_TABLES/normalization_log.csv`

## Формат лога нормализации

```csv
file,row_index,field,original_value,normalized_value,change_type,note
```

`change_type`: `unit_conversion` / `reference_filled` / `status_corrected` / `conflict`

## Поведение при конфликтах

- Если в документе референс отличается от стандартного >20% — записать как конфликт, не перезаписывать
- Если значение кажется ошибкой ввода (например, 1250 вместо 125) — отметить флагом `suspect=true`, не изменять

## Финальная проверка

После нормализации убедиться:
- Нет пустых значений в колонке `unit`
- Все `reference_min` и `reference_max` заполнены
- Все `status` ∈ {normal, low, high, text, unknown}
