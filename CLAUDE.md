# Health Dashboard — Главный файл инструкций

## Контекст проекта

Локальная система для хранения и анализа медицинских данных одного человека.
Все данные остаются на компьютере. Никаких облачных сервисов для хранения персональных данных.

## ⛔ КРИТИЧЕСКОЕ ПРАВИЛО: ДАШБОРД

**НИКОГДА не запускать `generate_dashboard.py` и не перезаписывать `05_SITE/index.html` через скрипт.**

`05_SITE/index.html` — это самостоятельный дашборд с ручным дизайном. Запуск `generate_dashboard.py` полностью перезаписывает его старым шаблоном и ломает всё.

Все изменения дизайна, данных и логики — **только прямое редактирование `05_SITE/index.html`** через Edit/Write.

Шаг «Дашборд» в рабочем процессе ниже — **устарел и не применяется**.

## Общие правила

- **Конфиденциальность:** Все данные обрабатываются локально. Никогда не отправлять медицинские данные в облако без явного разрешения пользователя.
- **Язык:** Русский (документы могут быть на русском или английском).
- **Единицы измерения:** Приводить к SI — ммоль/л, г/л, мкмоль/л, нмоль/л и т.д.
- **Дисклеймер:** При любых выводах и гипотезах обязательно указывать:
  > ⚠️ Это не медицинский совет. Все наблюдения носят информационный характер. Проконсультируйтесь с врачом.

## Структура папок

```
00_INBOX_RAW/       — сюда кидаются сырые документы
01_SORTED/          — отсортированные по типу и дате
02_TABLES/          — CSV/JSON с извлечёнными показателями
03_AGENT_REPORTS/   — отчёты агентов-специалистов
04_ADVISOR_SUMMARY/ — итоговые сводки
05_SITE/            — локальный HTML-дашборд
CLAUDE/agents/              — роли агентов
CLAUDE/prompts/             — шаблоны промптов
CLAUDE/prompts/complaints/  — жалобы для консилиума (читать перед запуском агентов)
scripts/            — Python-скрипты
```

## Формат таблиц (02_TABLES/)

Файлы CSV с колонками:

| Колонка | Описание |
|---------|----------|
| `date` | Дата анализа в формате YYYY-MM-DD |
| `test_name` | Название показателя (на русском) |
| `value` | Числовое значение |
| `unit` | Единица измерения (после нормализации — SI) |
| `reference_min` | Нижняя граница нормы |
| `reference_max` | Верхняя граница нормы |
| `status` | `normal` / `low` / `high` / `unknown` |
| `source_file` | Имя исходного файла |

### Файлы по типам:

- `blood_general.csv` — общий анализ крови (ОАК)
- `blood_biochemistry.csv` — биохимия крови
- `hormones.csv` — гормоны
- `vitamins.csv` — витамины и микроэлементы
- `urine.csv` — анализы мочи
- `other.csv` — прочие показатели

## Консилиум — правило жалоб

**Перед каждым консилиумом (запуском агентов-специалистов) обязательно:**

1. Найти последний файл в `CLAUDE/prompts/complaints/` (по имени, формат `YYYY-MM-DD_*.md`).
2. Прочитать его полностью — он содержит текущую жалобу, симптомы и вопрос для анализа.
3. Использовать этот файл как **основание и фокус** всего консилиума: каждый агент отвечает именно на эту жалобу, а не делает общий обзор данных.

Если файлов нет — спросить у пользователя, с какой жалобой проводится консилиум, и предложить создать файл.

### Формат файла жалобы

```
CLAUDE/prompts/complaints/YYYY-MM-DD_краткое-описание.md
```

```markdown
# Жалоба: [краткое название]

## Текущая жалоба / симптомы

[Описание от первого лица — что беспокоит, как давно, динамика]
```

## Рабочий процесс

1. **Сортировка:** Запустить агента `sorter` для файлов из `00_INBOX_RAW/`
2. **Извлечение:** Запустить агента `extractor` для файлов из `01_SORTED/`
3. **Нормализация:** Запустить агента `normalizer` для таблиц из `02_TABLES/`
4. **Анализ:** Запустить агентов-специалистов
5. **Дашборд:** редактировать `05_SITE/index.html` напрямую. ⛔ НЕ запускать `generate_dashboard.py` — перезапишет дашборд.

## Нутрициолог

Агент описан в `CLAUDE/agents/nutritionist.md`. Два режима работы:

- **В консилиуме (этап 1):** даёт краткий анализ жалобы по своей специализации — питание, нутриенты, дефициты, ЖКТ-контекст. Формат — стандартный для этапа 1 консилиума.
- **Отдельный запрос на ротацию меню:** работает по полному циклу из `CLAUDE/prompts/nutritionist_workflow.md`:
  1. Пользователь создаёт запрос в `08_NUTRITION/requests/` (шаблон: `08_NUTRITION/requests/template_nutrition_request.md`)
  2. Нутрициолог читает запрос + актуальные данные из `08_NUTRITION/` и `02_TABLES/`
  3. Генерирует ротацию в `08_NUTRITION/plans/`
  4. После фидбека обновляет `08_NUTRITION/logs/nutrition_journal.md` и `08_NUTRITION/food_profile.csv`

Папка нутрициолога: `08_NUTRITION/` — запросы, ротации, дневник питания, симптомы, профиль блюд, инвентарь.

### Правило: нутрициолог читает все данные автоматически

При каждом запросе нутрициолог **автоматически** читает:
- Все CSV из `02_TABLES/` (blood_general, blood_biochemistry, vitamins, hormones, genetics, medications — не нужно перечислять явно)
- Анамнез из `06_MEDICAL_HISTORY/` (gastroenterology.md и другие релевантные файлы)
- Текущий запрос из `08_NUTRITION/requests/`
- Профиль блюд и ингредиентов из `08_NUTRITION/`

Пользователь не обязан указывать список файлов — нутрициолог читает их самостоятельно.

### Правило: план питания → дашборд

**После каждого созданного плана питания** обязательно:
1. Добавить блок в `05_SITE/index.html` → раздел `sec-nutrition` (сразу перед закрывающим `</div>` секции).
2. Использовать паттерн `mh-accordion` / `<details>`:
   ```html
   <div class="mh-accordion">
     <details>
       <summary>YYYY-MM-DD · ключевое слово</summary>
       <!-- ключевые данные плана: фаза, цель, ккал, 5-дневная сводка, пометки -->
     </details>
   </div>
   ```
3. Заголовок `<summary>` = дата запроса + ключевое слово плана (1–2 слова).
4. Внутри — только ключевые данные (фаза, цель, ккал/день, краткая таблица 5 дней, ЖКТ-пометки). Полный план хранится в `08_NUTRITION/plans/`.
5. ⛔ Не запускать `generate_dashboard.py`.
